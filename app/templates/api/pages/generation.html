This section covers how to use the API to produce images and manipulate existing images. Please note that these endpoints do not actually return any data and are only to be used in reference to images, meaning you should use these URLs as if they are an existing image rather than making a GET request to one of them.

/text

Required arguments: text
Example URL: /text?text=Hello world!

This API endpoint takes the query argument text and returns an image with that text in the middle of it. If the text is long, the API will adjust the font size accordingly.

/badge

Required arguments: title, description, color
Example URL: /badge?title=Example title&description=Example content&color=#126bf1

This API endpoint will construct a "badge" using the given title, description and color. If no color is specified, then by default black will be used. The color can also optionally start with a hashtag although it is not required.

This endpoint will turn any valid unicode emoji, provided in the emoji parameter, into a valid PNG image via the twemoji CDN service. If the provided emoji is omitted or invalid, the endpoint will return a 400 error respectively.
